File Handling

Steps in File Handling

  • Reading From Keyboard
  • Writing in file
  • Reading from file
  • Write on screen
  • For only read from file, class ifstream
  • For only write in file, class ofstream
  • For both reading and writing in file, class fstream

For opening a file

  1. Opening a file for writing

    2. syntax-:ofstream object name("file name which you want to open.extension");
    When we open a file in write mode-:if that file is not existing, it create a new file of that name automatically.

  2. Opening a file for reading

    3. syntax-:ifstream object name("file name which you want to open.extension");
    When we open a file in read mode-:if that file is not existing, this class is not working. If file exits, it replace previous content with new content.

    • Text File

      • Files which are in human readable form. By default, file is considered as text files. Extention-:".txt"

    • Binary File

      • Files which are not in human readable form. Extension-:".dnd"

      Example

      ofstream X("ABC.txt");//For writing only
      int a;
      cin>>a;
      x< x.close();//close a file
      ifstream X("ABC.txt");//for reading only
      x>>a;

      File Operation modes

      Modes Description
      ios::in For reading(Default for reading)
      ios::out For writing(Default for writing)
      ios::binary For binary files
      ios::trunc deleting content of file
      ios::app add new data at end of the file(save previous data + new data)
      ios::ate file pointer point at end of the line(for both reading and writing)
      ios::nocreate does not create a new file when you open a file for writing and file is not exist
      ios::no replace does not replace the previous content of the file when you open a file for writing and file exits
